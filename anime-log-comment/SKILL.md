# Bangumi Anime Log Comment

进入本子 skill 前，先读：
- `../references/general-writing-layer.md`
- `references/writing-style.md`

## 适用任务

- 动画总评 / 单季长评 / 剧场版长评
- 单集延伸日志
- 角色关系分析
- 先梳理分集剧情，再写分析
- 把零散观感整理成站内长文

## 默认写法

1. 先判断类型：总评 / 单集 / 角色线 / 剧情整理
2. 素材不足时可查：作品简介、分集标题与范围、角色关系、公开剧情摘要
3. 开头直接给总判断，或交代本篇要处理的剧情切口
4. 第一部分抓住作品最成立的一点：主题、角色关系、演出策略、节奏或情绪组织
5. 第二部分展开论证：角色、叙事结构、镜头 / 演出 / 配乐 / 节奏如何支撑这个判断
6. 第三部分回到个人观感与边界：哪里被打动，哪里不成立，留了什么余韵
7. 结尾收束到“这部动画最后留下了什么”

## 输出要求

- 保留观感语言，但不碎成弹幕堆积
- 少写流水账，多写“为什么这点有效 / 无效”
- 单集日志可以更短，但必须有清晰判断和展开
- 只有公开简介级素材时，只写简介级分析
- 成稿默认交付 Markdown + Bangumi BBCode 两个版本；BBCode 版复用 `../scripts/markdown_to_bangumi_bbcode.py`
