# 动画日志写作层

基于 `data/anime/summary.md` 的 100 篇合格样本总结。

## 样本给出的明确倾向

- 动画日志常见开头不是背景介绍，而是：判断、问题、观感、单集切口、访谈 / 资料引子。
- 大量成稿会把“角色关系 / 主题判断”和“演出 / 节奏 / 结构”并列处理，而不是只谈其中一边。
- 合格日志非常重视余韵：即便正文分析很多，结尾也常回到观感为什么被留下。
- 小标题使用率高，但不是硬性要求；它通常出现在要同时分析主题、角色、演出和个人感受时。

## 推荐结构

### 1. 开头
优先从下面几种方式中选一种：
- 先给总判断：这部作品成立 / 不成立的核心是什么
- 从单集 / 单幕切入：借一个片段引出整部作品的问题
- 从观感反差切入：原本以为会怎样，结果真正打中的点是什么

### 2. 主体展开
建议按下面顺序挑 2 到 4 个维度推进：
- 角色与关系
- 主题与命题
- 演出、镜头、配乐、节奏
- 叙事结构与情绪递进

### 3. 结尾
回到：
- 这部动画最后留下了什么
- 这份感受为什么不是看完就散
- 它适合谁，不适合谁

## 语气要求

- 可以带个人口吻，但不要写成弹幕堆积
- 可以锋利，但要给理由
- 可以有诗意，但别把论证写没了

## 常见问题

- 只复述剧情，不做判断
- 一味夸作画 / 演出，却说不出为什么成立
- 一味讲主题，却跟具体动画表达脱节
- 结尾突然收不住，只剩一句“总之推荐”
