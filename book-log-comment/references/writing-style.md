# 书籍日志写作层

基于 100 篇 Bangumi 合格书籍日志样本的总结。

## 素材来源

写作前可参考以下素材（按优先级）：

1. **用户提供的笔记/摘录** → 优先使用
2. **EPUB 原文** → `scripts/read_epub.py` 提取
3. **用户给定的网站** → `scripts/fetch_web_content.py` 爬取书评、维基、豆瓣等
4. **Bangumi 其他用户长评** → `scripts/fetch_bangumi_logs.py` 抓取参考
5. **公开信息** → 作品简介、章节范围、角色信息

## 剧情框架完善

当用户要求"完善剧情框架"或"参考其他用户评论"时：

1. 调用 `fetch_bangumi_logs.py` 抓取 5-10 篇长评
2. 提取以下信息：
   - 剧情梳理角度（时间线、角色线、主题线）
   - 关键章节/段落的解读差异
   - 不同用户注意到的意象、伏笔、叙述诡计
3. 综合多篇长评的框架，形成自己的结构
4. **不要照搬**：参考角度，但用自己的语言和判断重写

## 样本倾向

- 书籍日志更容易“读后判断先行”，而不是先交代作者、出版信息或剧情简介
- 文本分析密度更高：主题、叙述、人物心理、结构安排、小标题切分都更常见
- 允许更强的概念和术语，但站得住的文章一定会落回文本细节
- 高质量样本会把“这本书让我怎么想”和“作者实际上怎么写”并在一起处理

## 推荐结构

### 1. 开头

从下面任选一种：

- 直接给读后判断
- 从不适、震动、惊喜、厌烦等阅读体验切入
- 从一个意象、角色、句子、章节切口切入

### 2. 主体展开

从下面挑 2 到 4 个维度推进：

- 文本气质 / 语言风格
- 主题与命题
- 人物与关系
- 章节 / 结构 / 叙述策略
- 个人理解与阅读边界

### 3. 结尾

回到：

- 这本书究竟值不值得被记住
- 它的问题到底是缺点，还是作者主动选择
- 你和它真正发生关系的地方在哪

## 语气要求

- 允许更高的分析密度，但别写成论文腔堆砌
- 可以有立场，但要有文本依据
- 可以更私密，但不能只有抒情没有判断

## 常见问题

- 把内容简介当评论主体
- 过度借外部理论，反而看不见文本本身
- 一味抒情，最后什么都没说清
- 概念过重，缺少落地细节
