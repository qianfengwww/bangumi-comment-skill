# Bangumi Game Log Comment

进入本子 skill 前，先读：
- `../references/general-writing-layer.md`
- `references/writing-style.md`

## 适用任务

- 通关日志 / 路线感想
- 玩法系统评价
- 视觉小说 / AVG / CRPG / 动作游戏长评
- 剧情梳理与路线分析
- 角色线 / 周目结构 / 系统与文本关系分析

## 默认写法

1. 先判断类型：通关总评 / 路线分析 / 剧情整理 / 系统评价
2. 素材不足时可查：作品简介、路线 / 章节范围、公开剧情摘要、核心系统说明
3. 开头先给通关后的总判断，或交代本篇要处理的路线 / 剧情范围
4. 第一部分抓玩法、系统、文本、路线结构里最核心的成立点或问题点
5. 第二部分展开到流程节奏、难度曲线、角色 / 路线 / 系统反馈
6. 第三部分回看整体：值不值得推荐，适合谁，不适合谁
7. 结尾收束到“这段游玩经历 / 这条路线最终留下了什么”

## 输出要求

- 可保留个人游玩记录感，但不写成纯攻略
- 系统、关卡、路线、文本要和主判断发生关系
- 涉及剧透时先提醒再展开
- 只有简介级素材时，只写简介级分析
- 成稿默认交付 Markdown + Bangumi BBCode 两个版本；BBCode 版复用 `../scripts/markdown_to_bangumi_bbcode.py`
