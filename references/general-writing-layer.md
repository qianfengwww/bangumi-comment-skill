# Bangumi 通用写作层（初版）

这份通用层负责把 Bangumi 长日志写得更像站内成稿，而不是公众号稿、说明书或流水账。

## 适用范围

- 动画日志
- 书籍日志
- 游戏日志

## 不负责

- 替用户捏造亲身体验
- 杜撰未提供的剧情、设定、路线或章节细节
- 把内容写成营销文或平台运营文案

## 默认起稿顺序

1. 先给一个明确判断，不要上来先讲百科背景。
2. 开头 2 到 4 段内要让读者知道：你为什么对这部作品有话要说。
3. 正文按“判断 → 展开 → 例子 / 场景 / 细节 → 回扣主旨”推进。
4. 每一大段只推进一个主问题，不要把剧情复述、价值判断、个人经历全塞进同一段。
5. 结尾回到“这部作品最后留下了什么”。

## 推荐骨架

- 标题
- 开头：快速抛出总判断 / 阅读或观看或游玩后的核心感受
- 第一部分：作品最抓人的点是什么
- 第二部分：这个判断为什么成立，展开到结构、角色、主题、系统或文本层
- 第三部分：个人体验、情绪余韵、适用边界或缺点
- 结尾：回收全文，让评价落地

## 行文规则

- 先说判断，再说论证
- 少做剧情流水账，多写“为什么这点有效 / 无效”
- 允许保留个人口吻，但不要碎到像聊天记录
- 每段尽量只讲一个意思
- 段落之间要有过渡，不要硬切
- 不要为了显得深刻而堆空话
- 不要为了像评论而故意端着

## 常用过渡句

- 先说结论
- 问题也出在这里
- 更具体一点
- 真正让我在意的是
- 如果把这个感觉拆开看
- 但这还不是全部
- 也正因为如此

## 通用质检

- 标题是否有明确落点，而不是空泛抒情
- 开头是否尽快给出判断
- 是否至少有 3 个自然段的有效展开
- 是否有具体细节支撑判断
- 结尾是否能收住，而不是突然中断
